# dbt Project Structure

Every dbt project has the same folder structure. Here is what each piece does.

## Folder Overview

```
learning_dbt/
├── dbt_project.yml       ← brain of the project
├── models/               ← your SQL transformation files (90% of your work)
├── seeds/                ← CSV files you want to load into Snowflake as tables
├── tests/                ← custom SQL tests you write to validate your data
├── macros/               ← reusable SQL logic (like functions) — advanced
├── analyses/             ← ad-hoc SQL you want to version control but not build
├── snapshots/            ← track how data changes over time — advanced
├── target/               ← auto-generated by dbt when you run it, never edit
└── logs/                 ← auto-generated log files, never edit
```

## Key Files

**`dbt_project.yml`**
The main config file. Tells dbt:
- The project name
- Which profile to use to connect to Snowflake
- Where to find each type of file
- Default settings for models (e.g. materialization)

**`~/.dbt/profiles.yml`**
Lives *outside* the project folder. Stores your Snowflake credentials.
Kept separate so you never accidentally commit passwords to Git.

## The Most Important Folder: `models/`
Every `.sql` file inside `models/` becomes an object in Snowflake when
you run `dbt run`. This is where you will spend most of your time.
